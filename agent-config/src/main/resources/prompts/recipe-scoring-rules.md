# 食谱营养评分规则

## 一、评分原则

### 核心理念
**评分基于用户实际配置的餐次进行评估，而非强制要求完整餐次配置。**

- 只配置早餐且符合营养要求 → 100分
- 配置早餐+午餐且符合要求 → 100分  
- 配置两天晚餐且符合要求 → 100分
- 配置完整三餐三点且符合要求 → 100分

### 评分逻辑
评分 = 各评分维度得分的加权平均值

**关键规则**：
1. 只评估**已启用且有菜品**的餐次
2. 推荐量按**已配置餐次的餐次占比**折算
3. 每个维度按**达标程度**评分（0-100分）

---

## 二、评分维度及权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 能量供给 | 30% | 热量是否在推荐范围内 |
| 宏量营养素配比 | 25% | 蛋白质、脂肪、碳水化合物比例 |
| 微量营养素达标率 | 20% | 维生素、矿物质达标情况 |
| 食材多样性 | 15% | 食物种类是否丰富 |
| 餐次结构合理 | 10% | 各餐次能量分配是否合理 |

## 四、餐次能量分配标准

### 标准餐次占比（参考）
| 餐次 | 占比 |
|------|------|
| 早餐 | 28-32% |
| 早点 | 5-8% |
| 午餐 | 35-40% |
| 午点 | 5-8% |
| 晚餐 | 28-32% |
| 晚点 | 5-8% |

### 动态占比计算规则
当用户只配置部分餐次时，按比例重新计算：

**公式**：实际餐次推荐量 = 全天推荐量 × (该餐次标准占比 / 已配置餐次总占比)

**示例**：
- 只配置早餐+午餐：
  - 早餐实际推荐量 = 全天推荐量 × 30% / (30%+40%) = 全天 × 42.8%
  - 午餐实际推荐量 = 全天推荐量 × 40% / (30%+40%) = 全天 × 57.1%

## 五、评分计算方法

### 1. 能量供给评分（30分）

**计算公式**：
```
能量得分 = 100 - |实际热量 - 推荐热量中值| / 推荐热量中值 × 100
```

**评分区间**：
- 实际热量在推荐范围80%-120%：100分
- 实际热量在推荐范围60%-80%或120%-140%：70分
- 实际热量在推荐范围40%-60%或140%-160%：40分
- 实际热量 < 40% 或 > 160%：10分

### 2. 宏量营养素配比评分（25分）

**三大营养素供能比标准**：
- 蛋白质：12-15%
- 脂肪：20-30%
- 碳水化合物：55-65%

**计算方法**：
```
配比得分 = (蛋白质得分 + 脂肪得分 + 碳水得分) / 3
单项得分 = 100 - |实际比例 - 推荐比例中值| / 推荐范围宽度 × 100
```

### 3. 微量营养素达标率（20分）

**关键营养素清单**：
| 营养素 | 权重 |
|--------|------|
| 钙 | 20% |
| 铁 | 15% |
| 锌 | 15% |
| 维生素A | 15% |
| 维生素B1 | 10% |
| 维生素B2 | 10% |
| 维生素C | 15% |

**计算方法**：
```
微量得分 = Σ(各营养素得分 × 权重)
单项得分 = min(100, 实际摄入量 / 推荐量 × 100)
```

### 4. 食材多样性评分（15分）

**评分标准**：
| 单餐食材种类 | 得分 |
|--------------|------|
| ≥8种 | 100分 |
| 6-7种 | 90分 |
| 4-5种 | 75分 |
| 2-3种 | 50分 |
| ≤1种 | 20分 |

**食材分类（需覆盖）**：
1. 谷薯类
2. 蔬菜类
3. 水果类
4. 畜禽肉类
5. 水产类
6. 蛋类
7. 奶及奶制品
8. 豆及豆制品

### 5. 餐次结构合理性（10分）

**评分要点**：
- 每餐是否有主食 + 蛋白质 + 蔬菜：60%
- 食物搭配是否避免相克：20%
- 烹饪方式是否多样：20%

## 六、快速评分估算方法（供AI使用）

### 估算流程

**步骤1：识别就餐人群**
从上下文 `nutritionGuidelines.crowdName` 获取，映射到对应年龄段标准。

**步骤2：计算已配置餐次的总推荐量**
```
总餐次占比 = Σ(已配置餐次的标准占比)
实际推荐量 = 全天推荐量 × 总餐次占比
```

**步骤3：估算食谱营养值**
```
热量估算 = Σ(菜品热量) 或 Σ(食材热量 × 用量)
蛋白质估算 = Σ(食材蛋白质含量 × 用量)
脂肪估算 = Σ(食材脂肪含量 × 用量)
碳水估算 = Σ(食材碳水含量 × 用量)
```

**步骤4：计算达标率**
```
热量达标率 = 估算热量 / 实际推荐热量
蛋白质达标率 = 估算蛋白质 / 实际推荐蛋白质
...以此类推
```

**步骤5：估算评分**
```
如果 所有达标率在 0.8-1.2 之间：
    预估评分 = 85-100分
否则如果 所有达标率在 0.6-1.4 之间：
    预估评分 = 70-85分
否则如果 存在达标率 < 0.5 或 > 1.5：
    预估评分 = < 60分
否则：
    预估评分 = 60-70分
```

### 快速达标检查清单

✅ **能量达标检查**：
- 早餐：热量在推荐量的80%-120%
- 午餐：热量在推荐量的80%-120%
- 晚餐：热量在推荐量的80%-120%

✅ **结构达标检查**：
- 每餐有主食（谷薯类）
- 每餐有蛋白质来源（肉/蛋/豆/奶）
- 每餐有蔬菜（午餐和晚餐）
- 全天食材 ≥ 12种

✅ **微量营养素关注点**：
- 钙：奶制品、豆制品、绿叶蔬菜
- 铁：红肉、动物肝脏、绿叶蔬菜
- 维生素A：胡萝卜、肝脏、蛋黄
- 维生素C：新鲜蔬菜水果

## 七、常见问题与扣分项

### 严重扣分项（扣20-30分）
- 单餐只有一道菜
- 全天无蔬菜
- 全天无主食
- 热量严重不足（<50%）或严重超标（>200%）

### 中等扣分项（扣10-20分）
- 单餐缺少蛋白质
- 全天蔬菜种类 < 3种
- 油盐超标（油>30g/天，盐>5g/天）
- 食材相克（如菠菜+豆腐同餐）

### 轻微扣分项（扣5-10分）
- 食材种类略少（全天8-10种）
- 烹饪方式单一（全油炸/全蒸煮）
- 颜色搭配单调

## 八、评分示例

### 示例1：只配置早餐（初中生）

**配置**：
- 牛奶燕麦粥（300g）：纯牛奶200g、燕麦40g
- 杂粮卷（120g）：玉米面60g、小麦粉60g
- 鸡蛋（50g）

**计算**：
- 实际热量：约550kcal
- 只配置早餐，实际推荐量 = 2000kcal × 30% = 600kcal
- 达标率 = 550/600 = 91.7%

**预估评分**：85-95分 ✅

### 示例2：配置早餐+午餐（小学生）

**早餐**：牛奶面包
**午餐**：米饭、红烧肉、炒青菜

**问题**：
- 早餐缺少蛋白质（只有牛奶不够）
- 全天蔬菜量不足
- 食材种类约6种

**预估评分**：65-75分（需改进）

## 九、给AI的提示

生成食谱时，请遵循以下步骤：

1. **确认年龄段标准**：从上下文获取就餐人群，选择对应营养标准

2. **计算实际推荐量**：根据已配置餐次计算各营养素实际推荐量

3. **生成符合标准的食谱**：
   - 确保热量在推荐量80%-120%
   - 确保蛋白质、脂肪、碳水比例合理
   - 确保每餐有主食+蛋白质+蔬菜
   - 确保全天食材种类 ≥ 12种

4. **自评验证**：
   - 估算生成食谱的营养值
   - 计算达标率
   - 如果预估评分 < 85分，调整后重新生成

5. **返回结果**：
   - 返回食谱数据
   - 返回预估评分
   - 返回营养说明
