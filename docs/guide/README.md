# yunxi Agent Platform 用户指南

欢迎来到 yunxi Agent Platform 用户指南。本指南将帮助你从入门到精通，掌握这个企业级多 Agent 协作框架。

## 本书阅读指南

### 学习路径总览

本指南采用"理论+实践"的方式编写，建议按照以下路径学习：

```
第一阶段：入门（约1小时）
├── 01. 介绍          → 了解框架定位和价值
├── 02. 快速开始      → 动手搭建环境，运行第一个示例
└── 03. 核心概念      → 掌握 Agent、MCP、规则引擎等核心概念

第二阶段：进阶（约2小时）
├── 04. 架构设计      → 理解分层架构、SPI 机制
├── 05. 模块说明      → 了解各模块职责
├── 06. 配置指南      → 掌握配置体系
└── 07. 开发指南      → 学习如何开发自定义 Agent

第三阶段：实战（约1.5小时）
├── 08. 部署指南      → 生产环境部署
├── 09. API 参考      → 接口调用（按需查阅）
├── 10. 技能系统      → 开发自定义技能
└── 11. 最佳实践      → 设计原则与优化建议

第四阶段：精通（可选，约1.5小时）
├── 14. 智能子系统    → 自适应策略、自我改进
└── 15. A2A 协议      → 跨服务 Agent 协作
```

### 不同角色的阅读建议

| 角色 | 推荐阅读 | 目的 |
|------|----------|------|
| **架构师** | 01、03、04、05、08、14、15 | 理解架构设计，做出技术决策 |
| **后端开发** | 02、03、06、07、10、11 | 掌握开发技能，编写业务代码 |
| **运维工程师** | 02、06、08、09、12 | 部署运维，故障排查 |
| **产品经理** | 01、03、04 | 了解能力边界，规划产品功能 |
| **技术管理者** | 01、04、05、11 | 评估技术方案，制定规范 |

### 理论知识体系

本指南涵盖的软件工程理论：

| 理论领域 | 涉及章节 | 核心概念 |
|----------|----------|----------|
| **AI 理论** | 01、03、14 | AI Agent、多 Agent 系统、ReAct、MCP |
| **架构设计** | 04、05、08、15 | 分层架构、微服务、依赖倒置、SPI |
| **设计模式** | 04、07、10 | 适配器、门面、策略、状态机 |
| **软件工程** | 07、11 | SOLID、DRY、KISS、测试理论 |
| **分布式系统** | 08、15 | CAP、服务发现、负载均衡 |
| **DevOps** | 08、12 | 容器化、监控、故障排查 |
